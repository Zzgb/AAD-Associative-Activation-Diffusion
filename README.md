# AAD — Associative Activation Diffusion

> 一套新的知识架构：从底层存储模型到上层发散推理、到规则嵌入、到短长期双记忆系统——用向量关联图谱统一替代「向量数据库 + RAG 检索 + Prompt 拼接」的传统管线。当前 AAD 1.0 是验证这套架构的极简原型。

## 架构概述

AAD 重新定义了 LLM 与知识交互的方式。不再把知识当作文档片段存进向量库，而是组织成**节点 + 向量关联**的有向图谱。LLM 不被动接收检索结果，而是沿关联路径自主发散探索——数据库、索引、推理、记忆四层合一 ：
(目前测试模型是合一架构,我的构想是分层: 基建:抽象关联数据库 LLM:发散推理架构实现 数据库内容:索引 短/长期记忆 ----- 数据库内容就像"神经元"模型一样互相关联由AI"前额叶"进行发散推理检索使用)


```
传统管线:
  向量数据库(存) → Embedding(检) → Top-K(取) → Prompt拼接(塞) → LLM(答)
  问题: 存储-检索-推理三层割裂，无状态，无关联，Token 膨胀

AAD 架构:
  关联图谱(存+检合一) → LLM逐跳发散(推理) → 短期记忆(状态) → 长期记忆(持久)
  存储即索引，关联即检索路径，探索过程即推理链，镜像即记忆
```

| 层 | 传统方案 | AAD |
|----|---------|-----|
| **存储模型** | 向量数据库存文档片段，无关联 | 关联图谱：节点 + 双向向量关联，有向图 |
| **检索** | Embedding → Top-K，一次性平铺 | LLM 沿关联逐跳发散，每次只加载当前节点 |
| **推理** | LLM 被动接收检索结果，拼接推断 | LLM 自主决定下一步探索哪个关联方向 |
| **规则** | 写死在 system prompt，静态不变 | 规则节点嵌入图谱，和数据共享索引，动态命中 |
| **短期记忆** | 无，每轮重新检索 | 会话图谱：自动镜像已加载节点，跨轮复用 |
| **长期记忆** | 向量库本身，无结构化更新 | 持久化关联图谱，可增删改，推理链可回溯 |

## 为什么不是又一个 RAG 变体

RAG 优化的是「检索精度」——怎么搜得更准。AAD 改的是「知识组织形式」——把知识从碎片变成图谱。检索精度是表，关联结构是本。节点之间有了关系之后，LLM 不需要每次从头检索，沿着关联走就是推理。

**这个版本验证了什么**：
- 关联图谱替代向量数据库作为存储层 ✓
- LLM 沿关联逐跳发散推理 ✓
- 短期记忆自动镜像 + 跨轮复用 ✓
- 会话推理链保留为消息关联图 ✓
- 干扰项隔离（零关联节点不会被误触达） ✓



## 推理策略：问题类型决定遍历方式

AAD 不预设固定的检索流程。用户的每个问题被分解为概念词元，每个词元触发独立的工具调用——**同一轮内全部异步并行**，不排队等待。

关键的是**根据问题类型选择遍历策略**：

### 关系类：「A 和 B 什么关系」

目标是找到两端的共同节点。采用**双向 BFS**：从 A 和 B 同时发散，碰撞即停。

```
用户: "光线追踪和量子计算和黄仁勋什么关系"

词元: [光线追踪] [量子计算] [黄仁勋]

第 1 轮（并行）:
  aad_lookup("光线追踪") → 关联: [RTX硬件加速] [图形渲染]
  aad_lookup("量子计算")   → 关联: []   ← 死胡同，终止
  aad_lookup("黄仁勋")     → 关联: [创立NVIDIA] [CUDA] [GPU] [AMD]

第 2 轮（双向并行）:
  从光线追踪: aad_expand(RTX关联) → RTX
  从黄仁勋:   aad_expand(NVIDIA关联) → NVIDIA

第 3 轮:
  RTX 的关联: [NVIDIA首创 → NVIDIA]  ← 碰撞！NVIDIA 同时出现在两端
  aad_trace("光线追踪", "黄仁勋") 确认路径:
  → 光线追踪 ↔ RTX ↔ NVIDIA ↔ 黄仁勋 (3跳)

量子计算: 无关联，已在第1轮终止
```

### 探索类：「GPU 能做什么」

目标是沿正确方向展开，不是找交集。每个关联的 `reason` 字段就是方向牌。

```
用户: "GPU 能做什么"

第 1 轮:
  aad_lookup("GPU") → 关联: [图形渲染] [深度学习] [GeForce] [比特币] [RTX] [AMD]

第 2 轮（并行发散，按需停止）:
  aad_expand(图形渲染) → 光线追踪
  aad_expand(深度学习) → CUDA → 人工智能
  aad_expand(比特币)   → 已足够，停止
  aad_expand(RTX)     → 光线追踪（已到达，碰撞停）
  aad_expand(AMD)     → GPU（回到起点，停）

→ 综合: GPU 用于 图形渲染、深度学习、GeForce、比特币挖矿、RTX 光追
```

### 策略对比

| | 关系类 | 探索类 |
|---|---|---|
| 目标 | 找两端之间的路径 | 沿关联方向展开 |
| 策略 | 双向 BFS，中间相遇 | 各方向独立，按需深入 |
| 停止条件 | 碰撞即停 或 两端耗尽 | 该方向信息足够 或 死胡同 |
| 工具 | aad_trace 优先 | aad_expand + reason 定向 |

## 逐跳探索示意（单节点）

```
用户: "黄仁勋的 GPU 公司有什么产品"

第 1 步 → aad_lookup("黄仁勋")
         ← 节点: 黄仁勋，关联: [创立了 NVIDIA] [领导 GPU 发明]
         上下文占用: ~200 Token（一个节点）

第 2 步 → aad_expand(NVIDIA 关联)
         ← 命中 NVIDIA，关联: [CUDA] [GeForce] [RTX] [数据中心]
         上下文占用: ~200 Token（一个节点）

第 3 步 → LLM: "信息够了，不用再查"
         → 输出答案
         总上下文占用: ~400 Token（两个节点 + 指令）
```

**关键区别**：RAG 一次塞入 10 个片段（可能 3000 Token），AAD 每步只加载当前节点（~200 Token），LLM 自主决定何时停止。窗口没满，注意力不散，推理链可追溯。

## 架构

```
用户输入 → CLI → AADAgent.run()
  ├── 短期记忆 (SessionMemory) — 会话级，/quit 销毁
  │     ├── msg_N 消息节点（线性链）
  │     └── 长期记忆镜像（自动拷贝，内容可改写）
  │
  └── DeepSeek Chat API（工具调用循环，LLM 自主停止）
        ├── aad_lookup(name)           → 短期优先 → 长期 → 自动镜像
        ├── aad_expand(ref, top_k)     → 双索引搜索（短期 + 长期）
        └── aad_get_content(name)      → 获取节点完整内容
```

| 组件 | 说明 |
|------|------|
| **Agent LLM** | DeepSeek Chat API，工具调用原生支持 OpenAI function-calling 格式 |
| **嵌入** | 本地确定性 hash（SHA-256 → 归一化向量），零外部依赖 |
| **长期记忆** | `AADStore`：内存 HashMap + JSONL 持久化，跨会话保留 |
| **短期记忆** | `SessionMemory`：会话级知识图谱，自动镜像 + 推理关联，/quit 丢弃 |
| **向量索引** | FAISS `IndexFlatIP`（长短各一个独立索引） |
| **SDK** | `openai` Python SDK（`base_url` 指向 `api.deepseek.com/v1`） |

## CLI 命令

| 命令 | 功能 |
|------|------|
| `> 你的问题` | 输入查询，Agent 探索图谱并回答 |
| `/nodes` | 列出所有长期记忆节点 |
| `/session` | 查看短期记忆状态（消息数、镜像节点） |
| `/quit`, `/q` | 退出（短期记忆自动清除） |
| `--reseed` | 启动时强制重生种子数据 |
